*   **✅ API Key/Token 만료 여부 확인:**
    *   사용하는 모든 외부 시스템(예: Notion, Google 등)의 API Key와 Access Token이 만료되었는지 확인하고, 최신 키로 재발급받습니다. [근거: sessions/2026-05-12T06-52/secretary.md]
*   **✅ Git 인증 방식 확인:**
    *   HTTPS 방식이라면 Personal Access Token(PAT)을 사용하도록 전환했는지 확인합니다.
    *   SSH 방식이라면, Host Key가 정상적으로 등록되어 있는지 확인합니다. [근거: sessions/2026-05-12T06-44/secretary.md]
*   **✅ 권한 범위 재확인:**
    *   GitHub 리포지토리에 접근하는 계정이 `Read` 외에 `Write` 권한까지 가지고 있는지 확인합니다.

#### 💻 Step 2: 로컬 환경 설정 (Local Environment Setup)
개발 환경 자체의 오류를 정리하고 최적화하는 단계입니다.

*   **✅ 개발 환경 클린 설치/재설치 (필요시):**
    *   터미널 및 개발 툴이 최신 버전인지 확인하고, 불필요한 캐시나 설정을 삭제합니다.
*   **✅ 로컬 리포지토리 상태 확인:**
    *   현재 작업 중인 로컬 리포지토리의 브랜치와 커밋 기록이 최신 상태인지 확인합니다.
*   **✅ 시스템 재부팅:**
    *   모든 인증 정보 및 네트워크 설정을 정리하기 위해, 최종적으로 시스템을 재부팅합니다. [근거: sessions/2026-05-12T06-52/secretary.md]

#### 🚀 Step 3: 동기화 재시도 및 검증 (Sync & Verification)
모든 준비가 끝났다면, 순서대로 명령어를 실행합니다.

1.  **원격 저장소 연결 확인:**
    *   `git remote -v` 명령어로 원격 저장소 주소가 올바른지 확인합니다.
2.  **최신 정보 Pull:**
    *   `git pull origin main` (또는 메인 브랜치 이름) 명령어로 원격 저장소의 최신 내용을 로컬로 가져옵니다.
3.  **로컬 커밋 푸시:**
    *   `git push origin main` 명령어로 로컬의 변경 사항을 원격으로 전송합니다.
4.  **최종 검증:**
    *   GitHub 웹사이트에서 해당 리포지토리의 파일 내용과 로컬 파일의 내용이 완벽하게 일치하는지 육안으로 확인합니다.
